## Provenance and isolation

- Capture root: `$CAPTURE_ROOT` (disposable; absolute host path omitted).
- Source: current shared Kandev checkout, exercised through a worker-scoped E2E backend and real browser controls.

- Encoding metadata: `metadata/capture.json`
- Source bundle: `source/capture.spec.ts`, `source/encode.mjs`, `source/product-loop-camera-compact.mjs`, `source/qa.mjs`, `source/smoke.spec.ts`
- Reframe source: full 3840x2400 masters, never the previously cropped delivery files.
- Reframe profile: Landing `scripts/product-loop-camera.mjs` docs profile, 1.50x maximum zoom, cosine easing, and frame-by-frame pointer containment. The two task clips use identical focused start/end crops to keep the feature legible while excluding the fixture-only Files toolbar and host path.

## Commands

- Capture/encoding source and per-clip camera values are retained in the disposable source and metadata bundle listed above.
- Camera validation: `createCameraTrack({ pointerTrack, pointerSafeMargin, ... })` checks every 25fps output frame and blocks a crop that loses the pointer.
- Long pointer journeys widen before travel and ease back into the destination instead of panning a tight crop after the pointer has left it.
- QA: `node "$CAPTURE_ROOT/source/qa.mjs" --visual-pass --teardown-confirmed`
- Probe template: `ffprobe -v error -show_streams -show_format -of json <file>`
- Cadence template: `ffprobe -v error -select_streams v:0 -show_entries frame=best_effort_timestamp_time,pkt_duration_time -of json <video>`
- Frame sampling: FFmpeg at opening, 10%, 25%, 50%, 75%, 90%, and ending timestamps; ImageMagick montage for contact sheets.
- Browser smoke: Chromium loaded each WebM, MP4, and WebP through a loopback HTTP server with byte-range support; both videos played and sought to 50%.
